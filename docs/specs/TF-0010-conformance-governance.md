## Conformance artifacts

Conformance should include test vectors, protocol traces, reference implementation tests, fuzzing corpus, interoperability suites, bridge tests, profile tests, proof verification tests, AI implementation tests, and security regression tests.

## Governance direction

TrustForge should begin as an open-source project and be structured so it can eventually become foundation-style or standards-body-compatible.

## Specification process

TrustForge documents should be RFC-style and versioned.

The Rust reference implementation and specification should evolve together.
