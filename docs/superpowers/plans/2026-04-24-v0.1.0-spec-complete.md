# TrustForge v0.1.0 Spec-Complete Implementation Plan

> **For agentic workers:** Execute task-by-task using checkbox (`- [ ]`) syntax for tracking. Each sprint must end green (TS tests pass, Rust tests pass, parity ok, codegen-diff clean) before advancing.

**Goal:** Implement every normative requirement from the TF-XXXX spec series, the bridge specs, the profile docs, and `DECISIONS.md` so v0.1.0 is genuinely spec-complete (Option A from the 2026-04-24 spec audit).

**Architecture:** Spec corpus is the source of truth. Each sprint takes a slice of the audit's 25 work areas, lands schemas + TS + Rust + parity vectors + tests + CI in one pass. No deferrals; hygiene (CHANGELOG, version bumps, README) is left for the final pre-release sprint so it isn't continually re-edited.

**Tech stack:** TS via Bun + `@noble/*`, Rust workspace with audited crates (`ed25519-dalek`, `chacha20poly1305`, `jsonwebtoken`, `x509-parser`, `ciborium`, etc.), JSON-Schema-driven codegen, byte-exact cross-language parity.

## Per-sprint completion bar

A sprint ends when:

1. All checkbox items above are marked done.
2. `bun run --filter '*' typecheck` is clean across the workspace.
3. `bun test` is green.
4. `cargo test --workspace` is green.
5. `bun run tools/tf-schema/src/cli.ts validate-all && lint && parity && fuzz --iterations 200` all clean.
6. `bun run tools/tf-schema/src/cli.ts codegen --target {ts,rust,docs,rpc-ts,rpc-rust,agent-contract-ts,agent-contract-rust} && git diff --exit-code` shows no drift.
7. CI workflow updated to exercise any new test entry points.
8. A single commit per sprint summarizing the work + Co-Authored-By footer.

## Quality bars baked in

- **No custom crypto.** Every primitive wraps an audited library.
- **TS + Rust parity for every wire-format type.** Byte-exact vectors in `conformance/`.
- **Spec + implementation move together.** Each sprint may amend the relevant TF-XXXX or `DECISIONS.md` if it discovers a gap; record diffs in the sprint commit.
- **No `#[allow(dead_code)]` shortcuts. No deferrals. No "ship the schema, wire it later".**

## Risk register

- **Native plugin sandboxing (E20)** could balloon ŌĆö plan B is to document "v0.1.0 native plugins are trusted; WASM is sandboxed" and ship a worker-thread isolate as the only sandbox boundary.
- **Hybrid PQ (C10)** depends on a usable ml-dsa Rust + TS implementation. Fallback: use only the audited `pqcrypto-mldsa` crate for Rust + the noble PQ extensions when stable; otherwise ship hybrid plumbing with a single PQ suite gated behind a feature flag.
- **gRPC bridge (E16)** introduces a `@grpc/grpc-js` dependency and HTTP/2 surface. Acceptable risk; alternative is to defer to v0.2 with explicit doc note ŌĆö but Option A says no deferrals.
- **Dashboard (F19)** could become a project of its own. Scope: viewer-only. No write actions through the dashboard for v0.1.0.

## Status tracking

Each sprint will have its commits tagged with `Sprint N:` prefix. Master plan owner updates the checkboxes above as sprints close. When all 25 areas are checked, cut `v0.1.0`.
