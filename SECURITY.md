# Security Policy

## Security posture

TrustForge is security infrastructure.

It must be treated as high-risk software.

## Cryptography rule

TrustForge must not casually invent new cryptographic primitives.

TrustForge should use reviewed cryptographic primitives and clearly defined crypto suites.

## Required security work

Before production use, TrustForge requires threat modeling, protocol review, cryptographic review, fuzzing, conformance tests, interoperability tests, misuse-resistance review, key-management review, plugin sandbox review, and formal analysis where possible.

## Experimental status

Initial TrustForge drafts and reference code are experimental until reviewed.
