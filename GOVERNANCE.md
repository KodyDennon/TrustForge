# Governance

## Initial model

TrustForge begins as an open-source project.

## Long-term direction

TrustForge should be structured so it can eventually support foundation-style governance, RFC-style specification process, independent implementations, conformance profiles, compatibility labels, and public security review.

## Spec process

Specs should be written as numbered TrustForge documents.

## Reference implementation

Rust is the flagship reference implementation.

Other implementations should be possible through conformance testing.
