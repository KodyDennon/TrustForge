# SELinux policy module for tf-daemon

The `trustforge.te` and `trustforge.fc` files together declare a confined `trustforge_t` domain plus the file-context mappings that route `/usr/bin/tf-daemon`, `/etc/trustforge`, `/var/lib/trustforge`, and `/run/trustforge/decide.sock` into it. To build and install on a RHEL/Fedora host: place both files in a working directory, run `make -f /usr/share/selinux/devel/Makefile trustforge.pp`, then `sudo semodule -i trustforge.pp` to load the module and `sudo restorecon -R -v /usr/bin/tf-daemon /etc/trustforge /var/lib/trustforge /run/trustforge` to relabel the filesystem. Verify with `getenforce` (must be `Enforcing`) and `ps -eZ | grep tf-daemon`. **Draft / experimental** policy; expect to add further `allow`/`dontaudit` rules from `audit2allow` once the daemon is exercised on a real host.
